For an overview on how to use fields, see this section. The accessors and type mappings reference is here.

This is a regular variant-like class that can represent any of the types that MySQL allows. It has value semantics (as opposed to field_view). Instances of this class are not created by the library. They should be created by the user, when the reference semantics of field_view are not appropriate.

Like a variant, at any point, a field always contains a value of certain type. You can query the type using field::kind and the is_xxx functions like field::is_int64. Use as_xxx and get_xxx for checked and unchecked value access, respectively. You can mutate a field by calling the emplace_xxx functions, by assigning it a different value, or using the lvalue references returned by as_xxx and get_xxx.

Description

Note that element accesors (like row_view::operator[]) return reference objects instead of value_type objects. You can use this type if you need an owning class.
