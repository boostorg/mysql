resultset::async_read_some (3 of 4 overloads)

Reads a batch of rows as a rows_view.

Synopsis 
template<
    class CompletionToken>
auto
async_read_some(
    use_views_t,
    CompletionToken&& token);
Description

The number of rows that will be read is unspecified. The contents of output will be replaced by the read rows, so you can obtain the number of read rows using output.size(). If this resultset has still rows to read, at least one will be read. If there are no more rows to be read, output will be empty().

The number of rows that will be read depends on the input buffer size. The bigger the buffer, the greater the batch size (up to a maximum). You can set the initial buffer size in connection::connect, using buffer_params::initial_read_buffer_size. The buffer may be grown bigger if required by other read operations.

The returned view points into the connection that *this references. It will be valid until the connection performs any other read operation or is destroyed.

This is the most complex but most performant way of reading rows. The handler signature for this operation is void(boost::mysql::error_code, boost::mysql::rows_view).
